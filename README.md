# Notification Service (Transactional Outbox)

Сервис отправки уведомлений с гарантией доставки, защитой от дублей и строгой типизацией.

## Технологический стек
- **PHP 8.3** (Strict types)
- **Laravel 12**
- **PostgreSQL 15** (с поддержкой SKIP LOCKED)
- **Redis** (для идемпотентности и очередей)
- **Pest** (тестирование)
- **PHPStan** (статический анализ, Level 5)

## Ключевые особенности
1. **Transactional Outbox**: Уведомления сохраняются в БД в одной транзакции с бизнес-логикой. Отдельный процесс (Relay) перекладывает их в очередь, что гарантирует доставку даже при падении Redis.
2. **Параллельный Relay**: Фоновый процесс спроектирован для запуска в несколько потоков. Использование `FOR UPDATE SKIP LOCKED` на уровне БД исключает конфликты и повторную обработку одного уведомления разными потоками.
3. **Идемпотентность**: Middleware проверяет заголовок `X-Idempotency-Key` и кэширует успешные ответы в Redis на 24 часа.
4. **Гарантия доставки**: Реализована политика повторов (Retry Policy) с экспоненциальной задержкой (backoff) в очередях.

## Установка и запуск
1. Поднять контейнеры:
   ```bash
   ./vendor/bin/sail up -d
   ```
2. Запустить миграции и сидеры:
   ```bash
   ./vendor/bin/sail artisan migrate --seed
   ```
3. Все процессы (API, Outbox Relay, Worker) настроены в `compose.yaml` и запускаются автоматически.

## Проверки

./vendor/bin/sail pest             # Тесты
